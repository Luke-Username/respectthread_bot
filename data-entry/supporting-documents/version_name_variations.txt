A document that keeps track of different verses and what regular expressions can be used to refer to them.
This file helps when using data_to_insert.py on characters that have not been added to the database yet.
The ctrl+f shortcut is typically used to find the relevant verse.
Note that these are guidelines, not rules.

Formatting
- Verses are ordered alphabetically
- Strings hold regular expressions (RegEx) that can be copy and pasted into Python or PostgreSQL
- "Version" and "Display Version Name" correlate to columns in the database (see: character_data.csv)
- Sometimes a number sign (#) is used to write additional comments
- An example of a verse is shown below

Name of the Verse
	Version
		'{"An array of RegExs to help tell what version of the character we need"}',
		'{"Both Version and Display Version Name are columns in the database (see: character_data.csv)"}',
		'{"Here is how you add apostrophe\'\'s to a verse name"}',
		r'{"Here is how you add \\(brackets\\) to a verse name"}'
	Specific (optional)
		'{"A RegEx that names a specific title of some verse. It is kept separate to help against false positives."}'
	Acronyms (optional)
		'{"Acronyms can be under Version, but sometimes they cause false positives if we are not careful"}'
	Sometimes Called (optional)
		'{"For verse names that might cause too many false positives if used with the character name"}'
	Other ways to determine the version (optional)
		'{"Any remaining patterns that do not match the other categories. Usually written as an if-statement."}'
	Displayed Version Name
		"This is shown when the bot replies to a post"
		# A verse can have multiple different display version names for various reasons. Write them underneath if needed.
			"2nd Displayed Version Name"
